# 인터뷰 12명 누적 발견 (v5 이전)

> PRD-v5.md §4에서 이관·구조화. 원문은 PRD 참조. v3=무빙, v4=파라디 표기.

## 세팅
- 대상: 유튜브 레시피를 실제로 따라 만드는 사용자 12명 (v1~v4 누적)
- 방식: 상상 기반 반응 위주. **실사용 관찰은 아직 미실행** (모든 반응이 상상 기반이라는 한계 명시됨).

## 핵심 발견
- **검증된 것**
  - 스텝 단위 카드 분할 + 하이라이팅은 요리 중 참고 관점에서 유효.
  - 저장 시스템 부재는 실제 페인포인트 (참가자들이 캡처·종이 메모·블로그로 자체 시스템 운영 중).
  - 아카이빙·카테고리 요구 반복 확인.
- **흔들리거나 미검증**
  - 손 물기 페인: 12명 중 5명만 명확 → 신뢰도 하락.
  - 재현 도구 포지셔닝: v3 결정 후 v4에서 미검증. 유즈케이스 정의가 12명 중 8명이 전부 다름.
  - 실사용 관찰 여전히 미실행.
- **근본적 재고 필요**
  - 카드 UI 자체: 무빙(v3)·파라디(v4)가 독립적으로 "전체 뷰 우선 + 카드는 상세 진입" 제안 → **H-VIEW**.
  - 임베디드 플레이어: v3 배제 → 파라디(v4) 강력 재요청 → **H-EMBED**.
  - 접근성 마찰: 4회 지적. 클립보드 감지만으로 부족 → **H-ACCESS**.
- **새로 부상한 방향**
  - 파라디(v4) 유연 검색 강화: "면 요리" 검색 시 파스타면·우동면 등 관련 재료 매칭. 재방문 트리거 후보 → **H-ACCESS**.

## 가설 영향
- `H-VIEW`: 미검증 유지 — 재고 필요 신호(무빙·파라디 독립 제안)가 v5 실험 동기.
- `H-EMBED`: 미검증 유지 — 파라디 재요청이 v5 실험 동기.
- `H-RETENTION`: 저장 페인 자체는 반복 확인(근거 확보), 재조회율 수치는 미검증.
- `H-ACCESS`: 미검증 — 마찰 4회 지적 + 유연 검색이 트리거 후보.
